# CLAUDE.md — Automation プロジェクト 絶対制約

このファイルはClaude Codeがこのプロジェクトで作業する際に常に従うべき制約を定義します。

## 絶対制約

### セキュリティ（最優先）
- `.env` ファイルは絶対に読み込まない（Claude自身がファイル内容を参照・出力しない）
- `.env` の内容をレスポンスや出力に含めない
- `config.yaml` には会社固有の情報（ドメイン・ID類）が含まれるため、内容をレスポンスや出力に含めない
- APIキー・トークン・パスワードを絶対にコードにハードコードしない
- ログにAPIキーや個人情報を出力しない
- 本番環境のデータを直接操作するコードは書かない

### 操作の確認
- 外部URL へのリクエストを伴うコードを実行する前に必ずユーザーに確認を求める
- ファイルの削除・上書きを行う前に必ずユーザーに確認を求める
- GitHub へ push する前に `.env` と `config.yaml` が `.gitignore` に含まれているか確認する

### 言語・環境
- Python バージョンは **3.11** で固定（`.python-version` 参照）
- パッケージ管理は **uv** のみ使用（pip直接実行禁止）
- 依存追加は `uv add <package>` で行い、`uv.lock` を必ずコミットに含める

### コード品質
- Linter/Formatter は **ruff** を使用
- コミット前に `uv run ruff check .` / `uv run ruff format .` を通す
- 型ヒントを積極的に使用する

### ディレクトリ構造
- 再利用可能なモジュールは `modules/` 配下に配置する
- エントリーポイントは `main.py` とする

### Git
- `main` ブランチへの直接プッシュは避け、機能ブランチを使用する
- コミットメッセージは日本語 or 英語どちらでも可
